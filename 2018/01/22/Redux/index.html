Redux — React Native

Posted on 2018-01-22

As we know, Web Application is a kind of state machine, where each view corresponds to a state. Based on this thought, Redux is introduced to store all state as a state manager.

  1. Store

    Store is a container holds all state. An application only has a store.

    createStore is the main function to create a Store.

  2. Action

    Action is the bridge connecting views and states, since users can’t get access to the states directly, they can only touch views, and call action to change the state.

    There are two important properties in the Action object:

    • type: the name of the Action
    • payload: the data transferred through this Action

  3. Action Creator

    Normally, we can create a function to generate an Action:

    store.dispatch()

    But how do we call an Action? Store provides a function to help us solve this problem, dispatch(). It received an Action object as an argument, and calls this function.

  4. Reducer

    After receiving an Action, Store has to change its corresponding state. The process is Reducer.

    Reducer takes Action and State as its arguments, and return a new state.

  5. connect() — ‘react - redux’

    React-Redux provides a connect() function to link UI Component and its business logic.

    • UIComponent is just a simple encapsulation of UI components

    • mapStateToProps — input logic , defines the way that transform external data to its props

    • mapDispatchToProps — output logic , defines the way that generate Action objects based on user’s operation, and send from the UI component.

      It construct a way to map the props of Component to store.dispatch.
